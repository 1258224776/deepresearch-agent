# DeepResearch Agent 发布短片设计

日期：`2026-04-23`  
项目：`D:\agent-one`  
产出类型：`Remotion 发布会风格产品视频`

## 1. 目标

为 `DeepResearch Agent` 制作一支 `75-90 秒` 的中文发布会短片。整体气质参考苹果新品发布视频，但内容必须建立在项目现有能力之上，而不是空泛品牌片。

这支片子的目标不是完整教学，也不是功能罗列，而是完成两件事：

1. 让观众先理解：这不是普通聊天工具，而是一个能完成研究任务的产品。
2. 让观众再理解：它为什么能做到这一点，包括规划、执行、记忆、汇总、引用，以及 URL 提取与可视化看板。

## 2. 已确认方向

以下方向已在设计沟通中确认：

- 视频时长：`75-90 秒`
- 成片类型：`发布会短片`
- 叙事重心：`前半讲产品价值，后半讲技术底座`
- 镜头策略：`混合型`
- 证据来源：`真实 UI + 真实流程`
- 气质来源：`概念化镜头 + 克制动效`
- 旁白：`中文男声`
- 产品名：`DeepResearch Agent`
- 成片方向：`产品发布片`
- 视觉方向：`C. Precision Workspace`

## 3. 项目事实基础

本设计只使用已经在项目中能成立的产品叙事，不引入未证实能力。

基于当前仓库内容，短片将围绕以下真实能力构建：

- `ReAct` 研究模式：对单个研究问题执行搜索、抓取、观察、继续推理，直至输出完整结果。
- `Planner` 多步调查模式：对复杂问题进行拆解、逐步执行，并最终综合结论。
- `Planner / Executor / Memory / Reporter` 的系统结构。
- `URL 智能提取`：从 URL 输入出发，经过并发抓取、结构化提取和看板汇总，形成强记忆点。
- 本地研究工作台：包含模式切换、输入区、会话区、执行轨迹、运行面板、引用与结果区域。
- 多模型路由与配置能力：不同引擎、不同 profile、不同研究路径。

短片不会承诺以下内容：

- 未在当前项目里落地的完整企业平台能力
- 不存在于仓库中的高度拟真 3D 产品建模
- 外部商业化指标、用户数据或市场成绩

## 4. 受众

这支片子采用混合型受众定位。

面向潜在用户时，重点是让他们快速理解：它能帮我完成研究，而不是只会聊天。  
面向开发者和技术合作方时，重点是让他们快速理解：它背后是一套有结构的研究系统，而不是单次 prompt 技巧。

因此，视频结构必须满足：

- 前半段直接给出产品价值与使用结果
- 后半段揭示系统机制与工程可信度

## 5. 创意原则

### 5.1 总原则

整片围绕一个主题展开：

> 研究，不该从混乱开始。

再落到一句收束语：

> 不是更快地聊天，而是更完整地完成研究。

### 5.2 风格原则

- 克制，不夸张，不赛博，不做蓝紫科技模板。
- 高级感来自秩序、构图、留白、文字、节奏，不来自复杂特效堆叠。
- UI 必须像一个真实产品，而不是模板化概念界面。
- 所有抽象动效都服务“从混乱到秩序”的视觉主题。

## 6. 叙事结构

建议时长：`82 秒`

### 6.1 0-10s 片头问题

目的：

- 从痛点切入，而不是先报产品名。
- 建立“研究工作当前很混乱”的感受。

画面：

- 黑场
- 零散标签页、链接、摘抄、碎片信息作为抽象元素短暂出现
- 最终被一条中心线收束

文案方向：

- 研究，不该从混乱开始

### 6.2 10-22s 产品登场

目的：

- 正式引出 `DeepResearch Agent`
- 让观众理解它是一个工作台，而不是一个普通聊天窗口

画面：

- 工作台整体结构作为英雄镜头进入
- 展示侧栏、主输入区、主内容区、运行面板的空间关系
- 慢推镜头建立产品质感

文案方向：

- 一个入口，进入完整研究流程

### 6.3 22-36s 产品价值

目的：

- 证明它不是直接吐答案，而是会执行研究流程
- 强调“联网搜索、抓取、轨迹、引用”是可见的

画面：

- 输入复杂问题
- 模式切换到研究或规划
- 执行轨迹、步骤流、引用感逐步出现

文案方向：

- 从问题开始
- 搜索、抓取、整理、引用

### 6.4 36-50s 核心亮点：URL 智能提取

目的：

- 形成整支片最强记忆点
- 把 README 中最容易视觉化的能力做成高潮段落

画面：

- 粘贴 URL
- 链接分流为并发抓取路径
- 多卡片抽取结果汇聚
- 汇总为结构化看板、统计卡片、趋势面板

文案方向：

- 从网页，到结构化信息
- 从链接，到可读结论

### 6.5 50-66s 技术揭示

目的：

- 从“它好用”过渡到“它为什么能做到”

画面：

- UI 退后
- `Planner / Executor / Memory / Reporter` 逐个出现
- 每个关键词伴随自己的视觉角色

说明方式：

- 不做底层代码讲解
- 只解释职责边界与系统协同关系

### 6.6 66-76s 多模型与工作方式

目的：

- 带出可扩展性与底座感

画面：

- 模式切换
- 引擎选择
- 运行面板
- 引用结果
- 最终结果卡片

文案方向：

- 不同问题，不同路径
- 同一工作台，完整闭环

### 6.7 76-82s 收束

目的：

- 用一句结论完成品牌记忆

画面：

- 工作台远景
- 品牌标题与收束文案
- 视觉能量下降，留下余味

文案方向：

- 不是更快地聊天
- 而是更完整地完成研究

## 7. 视觉系统

### 7.1 配色

采用 `Precision Workspace` 方向：

- 暖白
- 沙色
- 浅铜
- 深棕灰

黑场只用于：

- 片头
- 少量系统揭示转场
- 个别高对比收束镜头

### 7.2 画面层级

每个镜头保持三层结构：

1. `超大标题层`：负责气质、记忆点、发布会感。
2. `真实工作台层`：负责产品证据与可信度。
3. `抽象数据层`：负责流程、路径、信息组织的可视化。

### 7.3 镜头运动

允许的主要运动：

- 缓慢推进
- 横向滑移
- 局部放大
- 分层景深
- 高光扫过

避免：

- 花哨翻转
- 赛博粒子雨
- 快速剪辑堆叠
- 无意义镜头旋转

### 7.4 UI 使用原则

不直接使用原始整页录屏，而是对已有 UI 做“电影化重构”。重点提取：

- 输入框
- 模式切换
- 建议卡片
- 执行轨迹
- Run Inspector
- 引用与结果块
- Planner 节点结构

这些元素将作为可控组件进入视频画面，而不是被动录屏素材。

### 7.5 抽象元素边界

允许：

- 路径线
- 光带
- 浮层卡片
- 节点点亮
- 信息汇聚动画

不允许：

- 抢戏的科幻 HUD
- 偏离产品视觉的蓝紫赛博风
- 与项目无关的未来城市或芯片类隐喻

## 8. 旁白与声音

### 8.1 旁白

采用 `中文男声`：

- 低沉
- 克制
- 近似发布会主讲语气
- 不做表演式朗诵

文案风格：

- 短句
- 留白
- 大词少
- 动词清晰

目标不是覆盖全部功能，而是建立高密度认知锚点。

### 8.2 音乐

音乐方向：

- 低频合成器
- 极简钢琴或脉冲节拍
- 前半克制
- 中段抬升
- 尾段收束

节奏点：

- `36s` URL 智能提取开始抬升
- `50s` 系统揭示达到峰值
- `76s` 后收束

### 8.3 音效

音效只做轻量支持：

- 卡片落位
- 节点点亮
- 路径扫过
- UI 细微反馈

不能做成预告片式重击音墙。

## 9. 实现方案

### 9.1 技术方案

使用 `Remotion` 进行整片重建，而不是录屏剪辑。

原因：

- 统一构图与节奏
- 可控转场与标题系统
- 可替换文案、时长、语言
- 可输出封面、短版、英文版等衍生资产

### 9.2 预期工程结构

建议拆分为：

- `Root / index`
- 主 `Composition`
- `audioConfig` 或同类时序配置
- 各段 `Scene` 组件
- 公共 UI 组件
- 公共动效组件
- 视觉 token / 配色 / 字体配置
- 旁白、字幕、音乐资源目录

### 9.3 资产来源策略

资产优先级如下：

1. 项目真实产品信息
2. 项目现有前端结构抽象重建
3. 为视频新增的品牌化构图和抽象元素

不依赖：

- 外部不受控品牌素材
- 与当前产品视觉严重脱节的第三方模板

## 10. 交付物

建议最终交付：

1. `1080p MP4` 正片
2. 首帧或封面图
3. Remotion 源码工程
4. 可替换的中文旁白脚本

可选补充：

- 带配音版本
- 纯音乐版本
- 无字幕版本
- 英文文案版本

## 11. 验收标准

最终验收以以下三条为准：

1. 看上去像一个真实产品，而不是概念片。
2. 观众能在一遍内理解核心价值：`它完成研究，而不是只会聊天。`
3. 成片保留苹果式克制感，但没有落入通用科技宣传片模板。

## 12. 风险与缓解

### 风险 1：审美失控

表现：

- 太像模板
- 太赛博
- 太炫技

缓解：

- 先做 `10-15 秒` 风格样片验证
- 先锁色彩、标题、镜头速度，再扩全片

### 风险 2：信息过载

表现：

- 功能点过多
- 观众记不住

缓解：

- `URL 智能提取` 作为唯一绝对高潮
- 其他功能围绕研究闭环服务，不平均分配戏份

### 风险 3：过度依赖现有 UI

表现：

- 成片像录屏剪辑

缓解：

- 所有 UI 做镜头级重组，不使用原始整页录屏作为主素材

## 13. 里程碑建议

建议按以下顺序执行：

1. 建 `Remotion` 工程与基础 `Composition`
2. 先做 `10-15 秒` 风格样片
3. 锁定标题系统、配色与动效密度
4. 扩展为全片场景
5. 接入旁白与字幕
6. 输出正片与封面

## 14. 非目标

本次设计不包含：

- 完整品牌系统重塑
- 官网重做
- 当前前端产品代码的大规模 UI 重构
- 商业发行物料包

本设计只服务于本次 `DeepResearch Agent` 发布短片制作。
